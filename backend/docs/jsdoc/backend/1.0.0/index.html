ScanSew Backend

A modern Node.js backend for the Scan & Sew application, built with Apollo Server, Prisma, and PostgreSQL. This backend provides a GraphQL API for managing users, companies, work orders, and file storage.

Features

  • GraphQL API with Apollo Server
  • Authentication using JWT
  • File Storage with MinIO (S3-compatible)
  • Database with PostgreSQL and Prisma ORM
  • Type Safety with TypeScript
  • Containerized with Docker

Quick Start

  1. Clone the repository

    git clone https://github.com/yourusername/scansewapp.git
cd scansewapp/backend

  2. Install dependencies

    npm install

  3. Set up environment variables Copy .env.example to .env and update the values:

    cp .env.example .env

  4. Start the development server

    docker-compose up -d
npm run dev

  5. Access the GraphQL Playground Open http://localhost:4000/graphql in your browser

Development

Prerequisites

  • Node.js 16+
  • Docker and Docker Compose
  • PostgreSQL 13+
  • MinIO (included in Docker Compose)

Available Scripts

  • npm run dev - Start development server with hot-reload
  • npm run build - Build for production
  • npm start - Start production server
  • npm test - Run tests
  • npm run lint - Run ESLint
  • npm run format - Format code with Prettier

Database

This project uses Prisma as the ORM. To apply migrations:

npx prisma migrate dev

To open Prisma Studio:

npx prisma studio

File Storage

File uploads are handled by MinIO, an S3-compatible object storage service. The service is pre-configured in the Docker Compose setup.
